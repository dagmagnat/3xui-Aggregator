# Отображение лимитов и срока в Happ/подписках

Добавлено отображение лимитов трафика и срока действия подписки в клиентских приложениях.

## Что изменено

- В `/sub/:slug` и `/json/:slug` добавляется HTTP-заголовок `Subscription-Userinfo`.
  Клиенты, которые его поддерживают, показывают общую полосу трафика и дату окончания.
- В названия узлов добавляется понятная информация по конкретному региону:
  `Россия · 7/10 ГБ · ост. 3 ГБ · 15 дн.`
- Для каждого узла используется его отдельный лимит из поля `ГБ` возле узла.
  Если возле узла поле пустое, используется общий лимит клиента.
- При обновлении подписки агрегатор коротко опрашивает 3x-ui и берёт текущие `up/down` из `clientStats`.
  Если узел недоступен, используется кэшированная информация, а конфигурация не пропадает.
- В настройках панели появились переключатели:
  - показывать лимит и срок в названиях узлов;
  - отдавать `Subscription-Userinfo`;
  - обновлять расход трафика при каждом обновлении подписки.

## Важно

`Subscription-Userinfo` — глобальная строка для всей подписки. Поэтому точный лимит конкретного региона дополнительно выводится прямо в названии этого региона. Это нужно для сценария, когда, например, лимит задан только на Россию, а остальные узлы без лимита.

Если приложение не поддерживает полосу из заголовка, пользователь всё равно увидит лимит и остаток в названии узла.
