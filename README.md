# 3xui-Aggregator

Панель для объединения подписок и управления клиентами на нескольких 3x-ui узлах.

## Что уже есть

- отдельная админ-панель;
- добавление нескольких узлов 3x-ui;
- проверка доступности узлов;
- создание клиента сразу на нескольких узлах;
- единая ссылка подписки;
- просмотр карточки клиента и повторная выдача подписки;
- удаление клиента со всех узлов;
- импорт существующих клиентов с другого сервера 3x-ui;
- копирование клиентов между узлами;
- создание подписки для уже существующих клиентов.

## Быстрая установка одной командой

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/dagmagnat/3xui-Aggregator/main/install.sh)
```

Установщик на русском языке:

- спросит, ставить по IP или по домену;
- если выбран домен, поднимет HTTPS автоматически через Caddy;
- спросит логин и пароль панели;
- спросит порт панели, по умолчанию `3000`;
- проверит, свободен ли выбранный порт;
- клонирует проект в `/opt/3xui-aggregator`;
- запустит контейнеры через Docker Compose.

## Можно ли ставить на тот же сервер, где уже есть 3x-ui

Да, можно.

Но есть важные условия:

- для режима по IP нужен свободный порт панели, например `3000`;
- для режима по домену должны быть свободны порты `80` и `443`;
- если на сервере уже занят `80/443`, доменный режим этим установщиком не подойдёт без ручной донастройки.

## Работа с существующими клиентами

Теперь можно работать не только с новыми клиентами, но и с уже созданными в 3x-ui:

- можно импортировать клиентов с подключённого узла;
- импортированные клиенты отображаются в панели как обычные;
- можно выдать им подписку без пересоздания;
- можно скопировать клиента на другие узлы;
- при копировании создаются новые конфигурации на выбранных серверах.

Это позволяет подключить агрегатор к уже работающей системе без переноса пользователей.

## Текущие ограничения

Сейчас проект хорошо подходит для сценария:

- ты добавляешь узлы;
- создаёшь новых клиентов на выбранных узлах;
- получаешь одну общую подписку.

Импорт уже существующих клиентов из основного inbound 3x-ui как отдельный мастер пока не реализован.
Это следующий этап развития проекта.

## Ручной запуск без установщика

```bash
cp .env.example .env
nano .env
docker compose up -d --build
```

## Удаление

```bash
bash /opt/3xui-aggregator/uninstall.sh
```
