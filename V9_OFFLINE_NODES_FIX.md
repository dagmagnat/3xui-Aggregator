# v9: устойчивые подписки при недоступных узлах

Что изменено:

- `/sub/:slug` и `/json/:slug` больше не должны пропадать из-за одного offline-узла.
- Агрегатор сохраняет кеш настроек inbound каждого узла после успешного получения настроек.
- При генерации подписки используются локальные связи `client_nodes` и кеш inbound.
- Если узел временно недоступен, но его inbound уже был закеширован, регион остаётся в подписке.
- Если клиент был создан напрямую в 3x-ui с тем же UUID/login, агрегатор пытается добавить enabled-узлы как fallback, чтобы регион не пропадал из JSON/подписки.

Важно:

- Для уже существующих узлов кеш появится после первого успешного действия: Проверить узел, импорт/синхронизация, создание клиента или открытие JSON, когда узел online.
- Если узел ни разу не был успешно прочитан агрегатором, его невозможно корректно добавить в JSON без Reality/inbound параметров.
- Offline-узел может отображаться в клиенте, но пинг будет ошибкой/красным — это нормально.
